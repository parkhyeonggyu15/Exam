> 직원 2명 이상이 동시에 접속하여 채팅이 가능한 다중 클라이언트 시스템을 구현합니다.

## 📋 상세 요구사항

- **서버**: `ServerSocket(9999)` 바인딩 → `accept()` 루프 → Thread 기반 다중 클라이언트 브로드캐스트
- **클라이언트**: `Socket` 으로 서버 접속 → Swing UI(채팅창) 메시지 송/수신 연동
- **시연**: 직원(클라이언트) 2명 이상 동시 접속 채팅
- **예외처리**: `try-catch-finally` + `close()`

## 🎯 구현 순서 권장 (단계별 흐름)

> 정답 코드는 **SOLVED 프로젝트** 참고. 본 가이드는 단계별 절차만 안내.

### Step 1 — `ChatServer.main()` 의 흐름
1. `ServerSocket` 객체 생성 (포트 `PORT`)
2. 무한 루프 시작
3. 루프 안에서 `accept()` 호출 → 반환된 `Socket` 받기
4. 받은 Socket 으로 `ChatHandler` 인스턴스 생성
5. `clients` Set 에 추가 (브로드캐스트 대상)
6. 새 `Thread` 만들어 핸들러 실행
7. 종료 시 `finally` 에서 ServerSocket close

### Step 2 — `ChatServer.broadcast()` 의 흐름
1. `synchronized(clients)` 블록 안에서
2. for-each 로 모든 핸들러 순회
3. 각 핸들러의 `send(message)` 호출

### Step 3 — `ChatHandler` 의 3가지 책임
1. **생성자**: Socket 의 입출력 스트림을 BufferedReader / PrintWriter 로 감싸기
   - 한글 깨짐 방지를 위해 **UTF-8 명시 필수**
   - PrintWriter 의 두 번째 인자 `true` (autoFlush)
2. **run()**: readLine() 루프
   - 첫 줄을 닉네임으로 받아 저장 + 입장 broadcast
   - 이후 받는 줄마다 `nickname + " : " + line` 형태로 broadcast
   - finally 블록에서 컬렉션 제거 + 자원 해제 + 퇴장 broadcast
3. **send()**: out.println(message) 한 줄

### Step 4 — `ChatClient.connect() / send() / disconnect()`
1. **connect()**:
   - Socket 생성 → in/out 스트림 (UTF-8 명시)
   - 첫 줄로 닉네임 송신
   - `running = true` 설정
   - **Daemon Thread** 만들어 `readLoop()` 실행 (UI 닫혀도 자동 종료되게)
2. **send()**: out 이 null 이 아니면 println(message)
3. **disconnect()**: running=false → in/out/socket close (각각 try-catch)

> 💡 `readLoop()` 는 이미 구현되어 있음. 학생은 connect 안에서 Thread 만 시작하면 됨.

## 📌 자주 하는 실수

- ❌ `BufferedReader` 만 만들고 인코딩 미지정 → 한글 깨짐 (UTF-8 명시)
- ❌ `PrintWriter` 의 autoFlush(true) 옵션 누락 → 메시지가 안 가는 것처럼 보임
- ❌ `clients` 컬렉션을 동기화 없이 순회 → `ConcurrentModificationException`
- ❌ accept() 받은 Socket 을 메인 스레드에서 직접 처리 → 두 번째 클라이언트 접속 불가
- ❌ readLoop Thread 를 daemon 으로 안 만듦 → UI 닫혀도 JVM 안 죽음
- ❌ `finally` 에서 close 안 함 → 좀비 소켓 / 포트 점유
