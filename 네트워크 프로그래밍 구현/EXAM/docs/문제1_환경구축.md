# 🔵 문제 1. Java 개발환경 구축 및 GitHub 형상관리 (가이드)

> 본 문제는 코드 구현이 아닌 **환경 구축 결과물(스크린샷 + 설명)** 을 PPT에 정리하는 형태입니다.

---

## 📋 상세 요구사항

1. **JDK 설치** 후 IDE에 빌드 환경 구축, **MySQL JDBC Driver(jar)** 빌드패스 추가
2. **MySQL 서버 설치**, `BookDB` 생성, 원격 접속 계정(`dbconn`) 권한 부여
3. **GitHub Repository 생성**, clone / commit / push 흐름 시연, `.gitignore` 보안 처리

## 🧭 절차 가이드

### 1) JDK 설치 확인
```cmd
java -version
javac -version
echo %JAVA_HOME%
```
> JDK 17 이상 권장. 출력을 PPT 에 스크린샷.

### 2) MySQL JDBC Driver 추가
1. <https://dev.mysql.com/downloads/connector/j/> 에서 ZIP Archive 다운로드
2. 압축 해제 후 `mysql-connector-j-9.x.x.jar` 파일을 `Ex/lib/` 에 복사
3. (Eclipse) 프로젝트 우클릭 → Build Path → Add External JARs → 위 파일 선택
4. (IntelliJ) File → Project Structure → Libraries → `+` → 위 파일 선택

### 3) MySQL DB 생성
```cmd
mysql -uroot -p < db\schema.sql
```
또는 MySQL Workbench 로 `db/schema.sql` 실행.
스키마 파일에 이미 다음이 포함되어 있음:
- `CREATE DATABASE BookDB`
- `CREATE USER 'dbconn'@'%' IDENTIFIED BY 'dbconn1234!'`
- `GRANT ALL PRIVILEGES ON BookDB.* TO 'dbconn'@'%'`

### 4) DB 접속 정보 분리 (보안)
```cmd
copy config\db.properties.sample config\db.properties
notepad config\db.properties
```
- `db.password` 를 본인 비밀번호로 변경
- `.gitignore` 가 `db.properties` 를 제외하는지 확인:
  ```cmd
  git status
  ```
  → 목록에 `db.properties` 가 **나타나지 않아야** 정상

### 5) GitHub 흐름
```cmd
git init
git add .
git commit -m "chore: NCS 평가 프로젝트 초기 골격"

REM (GitHub 에서 빈 repo 생성 후)
git remote add origin https://github.com/<id>/<repo>.git
git branch -M main
git push -u origin main
```

PPT 에 다음 스크린샷 포함:
- GitHub Repository 메인 페이지
- `git log --oneline` 출력
- `.gitignore` 의 내용

## 📌 자주 하는 실수

- ❌ `db.properties` 를 그대로 커밋 → **체크리스트 1-5 0점**
- ❌ JDBC jar 를 src 폴더에 두고 `import` 시도 → 빌드패스에 추가해야 함
- ❌ MySQL 사용자에 권한만 주고 `FLUSH PRIVILEGES` 누락
- ❌ `bin/`, `*.class`, `*.jar` 를 통째로 커밋 → 저장소가 비대해짐
