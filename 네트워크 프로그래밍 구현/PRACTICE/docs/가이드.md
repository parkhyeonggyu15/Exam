## 🟦 Step 0 — 환경 구축 (체크리스트 능력단위요소 1, 20점)

### 0-1. JDK 설치 확인 (체크리스트 1-1, 3점)
```cmd
java -version
javac -version
```
> JDK 17 이상.

### 0-2. MySQL MemoDB 셋업 (체크리스트 1-2, 4점)
```cmd
mysql -uroot -p < db\schema.sql
```
- 확인: `SHOW DATABASES;` 에 `MemoDB` 존재
- 확인: `SHOW GRANTS FOR 'memouser'@'%';` 로 권한

### 0-3. JDBC Driver 추가 (체크리스트 1-3, 4점)
- <https://dev.mysql.com/downloads/connector/j/> 에서 ZIP 다운
- `mysql-connector-j-9.x.x.jar` 를 `lib/` 에 복사
- Eclipse 자동 인식 또는 우클릭 → Build Path → Add JARs

### 0-4. GitHub 흐름 (체크리스트 1-4, 4점)
- Repository 생성 → clone → commit → push 한 사이클

### 0-5. db.properties 분리 (체크리스트 1-5, 5점)
```cmd
copy config\db.properties.sample config\db.properties
```
- `git status` 로 `db.properties` 가 제외되는지 확인
- 코드 안에 비밀번호 하드코딩 절대 금지

## 🟢 Step 1~4 — 메신저 (체크리스트 능력단위요소 2, 30점)

### Step 1 — `MessengerServer.main()` (체크리스트 2-1, 6점)
1. `ServerSocket(PORT)` 생성 → 시작 로그
2. 무한 루프: `accept()` → `MessengerHandler` 생성 → `clients.add` → `Thread.start()`
3. finally 에서 ServerSocket close

### Step 2 — `MessengerHandler` (체크리스트 2-3, 2-4, 2-5)
1. **생성자**: in/out 스트림 (UTF-8 명시)
2. **run()**: 첫 줄 닉네임 → readLine 루프 → broadcast
3. **finally**: 자원 해제 + 컬렉션 제거 + 퇴장 알림

### Step 3 — `MessengerServer.broadcast()` (체크리스트 2-4)
- `synchronized(clients)` 안에서 for-each → `h.send(message)`

### Step 4 — `MessengerClient` (체크리스트 2-2, 2-5)
1. **connect()**: Socket → in/out → 닉네임 송신 → Daemon Thread 시작
2. **send()**: out.println
3. **disconnect()**: 자원 해제

## 🟡 Step 5~7 — 메모 CRUD (체크리스트 능력단위요소 3, 30점)

### Step 5 — `Memo` VO (체크리스트 3-1, 6점)
1. private 필드 4개: `memoId`(Integer), `title`(String), `content`(String), `createdAt`(Timestamp)
2. 전체 인자 생성자 본문
3. getter/setter 8개
4. toString()

> 💡 Eclipse `Alt+Shift+S` 단축키로 자동 생성 가능.

### Step 6 — `DBManager.getConnection()` (체크리스트 3-2, 6점)
1. `Properties` 로 `config/db.properties` 로드
2. `Class.forName(driver)` (JDBC 4단계 ①)
3. `DriverManager.getConnection(url, user, password)` (JDBC 4단계 ②)

### Step 7 — `MemoDAO` 4개 메서드 (체크리스트 3-3, 3-4, 3-5)

#### 공통 패턴 — 모든 메서드 동일
```
try (Connection conn = DBManager.getConnection();
     PreparedStatement pstmt = conn.prepareStatement(SQL_XXX)) {
    pstmt.setXxx(idx, value);     // ← 파라미터 바인딩 (SQL Injection 방어)
    return pstmt.executeUpdate(); // 또는 executeQuery() + ResultSet 처리
}
```

#### 4개 메서드 매핑

| 메서드 | SQL 상수 | 파라미터 |
|---|---|---|
| `insert(Memo)`         | `SQL_INSERT`      | (1) title  (2) content |
| `selectAll()`          | `SQL_SELECT_ALL`  | 없음 → ResultSet 순회 → Memo List 반환 |
| `update(Memo)`         | `SQL_UPDATE`      | (1) title  (2) content  (3) memoId |
| `delete(int memoId)`   | `SQL_DELETE`      | (1) memoId |

> SQL 상수는 이미 MemoDAO 안에 선언됨 — 그대로 사용.

## 🎬 시연 흐름

### 메신저 시연
1. `run-server.bat` → "[MessengerServer] 서버 시작 - 포트: 8888"
2. `run-chat.bat` 두 번 → 두 클라이언트 [접속] (서로 다른 닉네임)
3. 한쪽에서 메시지 → 양쪽 채팅창 모두 표시
4. 한쪽 [종료] → 다른 쪽에 "[퇴장] xxx" 표시

### 메모 CRUD 시연
1. `run-memo.bat` → 시작 시 자동 조회 (샘플 5건)
2. 제목/내용 입력 → [추가] → 목록 즉시 반영
3. 행 클릭 → 폼 자동 채움 → 내용 수정 → [수정] → 반영
4. 행 클릭 → [삭제] → 확인 → 목록에서 제거

## ❓ 자주 하는 실수

- ❌ `db.properties` 그대로 commit → 체크리스트 1-5 0점
- ❌ `Statement` 사용 → 체크리스트 3-4 0점
- ❌ `BufferedReader` 인코딩 미지정 → 한글 깨짐
- ❌ `PrintWriter` autoFlush=true 누락 → 메시지 송신 안 됨
- ❌ 클라이언트 readLoop Thread 가 daemon 아님 → UI 닫혀도 JVM 안 죽음
- ❌ `Memo` 필드 public → 체크리스트 3-1 감점
